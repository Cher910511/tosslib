## 页面概述

在 **软件库** 列表中点击软件名称，即可进入该软件的**软件详情页**。详情页全方位聚合了开源组件的元数据、安全漏洞、合规资质以及供应链深度画像，为团队在软件引入与漏洞应急排查时提供全量、高价值的决策依据。

页面顶部展示当前版本的状态：

* **可信度量**：展示当前版本号、Scorecard 安全评分、集成风险及软件生命周期成熟度等指标，用于综合评估软件的可信与治理水平；
* **高危风险**：实时统计当前版本已收录的漏洞数量，并对风险进行重点标识，帮助用户快速感知潜在安全问题；
* **操作**：页面右上角提供选型、订阅、下载按钮，支持用户将软件添加至个人或组织中。订阅后，可持续接收该软件的漏洞预警通知。

### 1. 版本信息

版本信息模块用于展示开源软件的基础画像与核心元数据，帮助用户快速了解项目背景、维护情况及参考指标。

- **README**：展示当前仓库主分支 README 的原文及对应版本译文内容，包括项目简介、官方网站、下载地址、文档中心、社区渠道（如 Discord）以及主流分支的构建与使用说明等信息。

- **右侧信息栏**：聚合展示软件的关键指标与基础属性，包括主编程语言、开发商、版权报告（Copyright）、代码量、集成风险、行业分类等参考信息，便于用户进行综合评估与选型分析。

![sl-detail-2](../images/sl-detail-2.png)

### 2. 依赖关系

依赖关系以结构化列表展示当前软件版本所引入的全部上游依赖组件，帮助用户快速梳理软件的上游组件引用关系及潜在供应链影响范围。

* 提取关键字段，包括组件名称、`groupId`、`PackageName`、版本号及编程语言等信息，支持全量展开、收起以及数据一键导出；

* 为每个上游依赖组件提供快捷入口，支持跳转至组件详情页或直接下载。

> * `groupId`：组件所属组织或发布方标识，通常用于区分组件来源。

![sl-detail-3](../images/sl-detail-3.png)

### 3. 漏洞信息

漏洞信息模块用于展示当前软件版本已收录的全部安全漏洞，帮助用户快速识别当前软件版本存在的安全风险及漏洞影响情况。

* 列表展示 `CVEID`、CVSS 评分、漏洞等级及发布日期等核心信息，并支持一键导出漏洞清单；

* **漏洞详情侧边栏** — 点击 `CVEID` 后，可在右侧抽屉查看漏洞详细信息，包括关联漏洞编号、补丁修复链接、安全问题以及最终受影响的组件。

> * `CVEID`：公开披露漏洞的标准编号，用于唯一标识一个安全漏洞。

> * `CVSS 评分`：通用漏洞评分标准（Common Vulnerability Scoring System），用于衡量漏洞风险等级，分值越高表示风险越高。其后标注的 `v3.1 / v4.0` 为 CVSS 评分模型的不同版本，用于体现评分规则与计算方式的迭代更新。

![sl-detail-4](../images/sl-detail-4.png)

### 4. 恶意代码

恶意代码模块用于识别软件源码包中潜在的恶意行为与供应链投毒风险。

* 自动化扫描并展示存在风险的文件名称、恶意类别、恶意类型及恶意类型小类等信息；

* 提供威胁等级与检测置信度评估，辅助安全人员快速定位高风险内容并进行研判。

![sl-detail-5](../images/sl-detail-5.png)

![sl-detail-6](../images/sl-detail-6.png)

### 5. SBOM

SBOM 模块用于展示软件中的完整组件清单，为供应链审计与资产治理提供标准化数据支撑。

* 规范展示 `groupId`、`packageName`、`version` 以及标准化的 **PURL** 唯一标识；支持企业进行组件追踪、依赖分析及自动化供应链管理。

![sl-detail-7](../images/sl-detail-7.png)

### 6. 贡献者信息

贡献者信息模块用于分析开源项目的社区活跃度与生态健康度。

* 通过图表展示贡献者的国家/地区及组织/公司分布情况；底部展示 **Scorecard 评分（满分 10 分）** 与 **Criticality（项目关键度）评分（满分 1 分）**，从开源治理与生态影响力维度评估项目可信度；

* `Scorecard` 评分：基于自动化标准的软件安全治理评估；
* `Criticality` 评分：衡量该软件在开源生态中的核心影响力与关键程度。

![sl-detail-8](../images/sl-detail-8.png)

### 7. 许可证信息

许可证信息模块用于识别软件所声明的开源许可证，并辅助企业进行合规风险判断。

* 展示软件使用的开源许可证类型（如 `Apache License 2.0`），并判断是否属于企业已批准的合规许可证；

* **许可证详情侧边栏** — 支持查看许可证详细条款，图形化展示协议中的「允许（Permitted）」「限制（Forbidden）」与「要求（Required）」内容，包括商业使用、代码修改、合规分发、版权声明保留等关键法务条款，帮助企业降低知识产权与合规风险。

![sl-detail-9](../images/sl-detail-9.png)

### 8. 软件版本

软件版本模块用于查看平台收录的软件历史版本信息，并支持多版本横向对比分析。

* 展示各历史版本的版本号、发布时间、开发商，以及对应版本的漏洞数量与 Scorecard 评分；

* 支持点击对应版本快速跳转至版本详情页，查看具体版本数据；帮助研发与安全团队在当前版本存在风险时，快速评估并选择更稳定、更安全的历史替代版本。

![sl-detail-10](../images/sl-detail-10.png)

> **最佳实践**
> 
> 1. **软件选型与引入前**：建议优先查看「许可证信息」与「恶意代码」模块，确认软件许可证符合企业合规要求，且源码包不存在潜在投毒或恶意行为风险。
>
> 2. **漏洞应急与版本排查时**：当收到漏洞预警后，可进入「漏洞信息」查看漏洞详情，重点分析其攻击向量、权限要求及用户交互条件，判断是否会对当前业务环境产生影响；同时结合「软件版本」模块，快速筛选已修复漏洞且风险更低的替代版本。
