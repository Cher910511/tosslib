# 需求反馈

进入 **需求反馈** 页面后，用户可提交在使用开源代码库过程中遇到的问题、优化建议或资源缺失请求。平台通过结构化表单收集反馈信息，并进行统一流转、评估与处理，以持续完善可信开源生态能力。

## 反馈表单配置说明

用户在填写反馈时，系统会根据所选“反馈类型”动态调整表单字段，以确保收集到完整且有效的上下文信息。

### 1. 通用基础必填字段

无论选择何种反馈类型，以下字段均为必填项：

- **联系方式**：支持手机号或邮箱，用于平台在处理反馈或需要进一步沟通时联系用户。
- **反馈内容**：文本输入框，最多支持 300 字，用于描述具体问题、使用场景或优化建议。

### 2. 反馈类型与动态表单联动逻辑

系统根据用户诉求提供以下分类，不同类型将触发对应的补充字段：

**类型 A：资源缺失**

当用户在平台中未能检索到某个开源软件、组件或历史版本时，可选择此类型进行反馈，平台将根据提交信息进行资源补全或引入评估。

- 缺失资源名称：填写未检索到的开源软件或组件名称（如 `spring-cloud-openfeign-docs`）。

- 缺失资源托管地址：提供该资源在 GitHub、AtomGit 或官方代码仓库的 URL，用于后续自动化引入与安全合规审查。

**类型 B：功能需求 / 使用体验 / 异常反馈 / 权限安全 / 账号登录 / 其他**

- 当用户针对平台功能优化、界面体验、系统 Bug、权限控制或登录问题提出反馈时选择此类型。

系统将根据具体反馈内容进行分类处理与优先级评估，并流转至对应产品或研发模块进行跟进。
