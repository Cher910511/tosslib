# 软件扫描

## 功能概述

软件扫描用于对已上传的软件版本与组件进行安全检测与风险分析。

平台支持对软件包或组件包进行扫描，识别其中的漏洞、恶意代码、许可证风险及依赖关系，帮助用户快速评估软件的安全性、合规性与可信度。

扫描能力包括：漏洞检测、恶意代码检测、License 合规分析、组件解析、依赖关系分析、SBOM 生成、风险等级评估等。

## 扫描列表

进入 `软件管理 - 软件扫描`，页面顶部提供 **软件** 和 **组件** 两个标签页，点击可切换扫描对象。

- 软件标签页展示所有用户新增的软件版本（按版本平铺）；
- 组件标签页展示所有用户新增的组件；

扫描对象均为用户新建时上传的软件文件或组件文件。

## 软件扫描

软件扫描列表字段包括：软件名称、版本、扫描时间、扫描状态、漏洞数、扫描操作。

支持按软件名称搜索，同一软件的不同版本会分别展示为独立行，仅支持对已上传文件的软件版本进行扫描。

![sm-scan-1](../images/sm-scan-1.png)

## 组件扫描

组件扫描列表字段包括：组件名称、组件版本、编程语言、开源许可证、扫描时间、扫描状态、扫描操作。

支持按组件名称搜索，同名不同版本组件会分别展示，扫描结果互不影响。

![sm-scan-1](../images/sm-scan-2.png)

## 开始扫描

在扫描列表中，通过左侧复选框选择待扫描的软件版本或组件后，点击列表顶部“开始扫描”按钮即可创建扫描任务。

创建成功后，系统提示：

```text
已成功创建 n 个扫描任务
```

随后扫描状态变为“扫描中”，扫描操作区域显示扫描进度，系统后台开始自动分析。

支持：单个扫描、批量扫描、当前页全选、当前页取消全选。

当选中条目后，页面顶部会显示：

```text
已选择 x 项
```
取消全部选择后提示自动消失。

![sm-scan-3](../images/sm-scan-3.png)

## 再次检测

对于已扫描过的条目（扫描成功或扫描失败），按钮显示为「再次检测」。

点击再次检测后：
- 按钮变为扫描进度条
- 状态切换为「扫描中”
- 扫描完成后新结果替代前一次结果

对于从未扫描过的条目：
- 扫描列表显示「开始检测」

## 扫描状态说明

扫描状态分为以下四种：

| 状态 | 说明 |
|---|---|
| 待扫描 | 尚未开始扫描 |
| 扫描中 | 当前正在扫描分析 |
| 扫描成功 | 扫描完成且无异常 |
| 扫描失败 | 扫描过程中出现错误 |

说明：
- 扫描成功与扫描失败均支持再次检测；
- 扫描失败通常与文件异常或解析失败有关；
- 扫描状态会实时更新；

## 查看扫描结果

未扫描的软件版本或组件不支持进入扫描结果详情页。

扫描成功后：

- 软件版本号会变为蓝色可点击状态
- 组件名称会变为蓝色可点击状态

点击后即可进入扫描结果详情页。

![sm-scan-4](../images/sm-scan-4.png)

## 常见问题

**Q：开始扫描后多久可以看到结果？**

A：扫描时间与以下因素有关：文件大小、依赖数量、组件复杂度、当前任务量。扫描过程中可在列表中实时查看扫描进度。

**Q：再次检测会覆盖之前的扫描结果吗？**

A：会。再次检测完成后，新的扫描结果会完全替代之前的结果。

**Q：为什么有些条目无法点击查看详情？**

A：仅“扫描成功”的条目支持进入扫描结果页。未扫描、扫描中、扫描失败状态不可查看。

**Q：全选是否包含所有分页数据？**

A：不会。全选仅作用于当前页面的数据。如需跨页批量扫描，请切换分页后分别进行选择。

**Q：扫描失败如何处理？**

A：建议优先检查上传文件是否完整、文件格式是否正确、文件是否损坏、文件是否超出大小限制。支持上传格式包括：、ZIP、JAR、WAR、TAR、工程镜像、容器镜像。
