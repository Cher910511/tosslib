# 软件列表

## 功能概述

软件列表用于管理用户所有的软件及其版本和组件。并支持对软件文件和组件文件进行安全扫描、漏洞分析与合规检测。

系统采用树形层级结构管理软件数据：

- 一个软件可包含多个版本
- 一个版本可包含多个组件
- 软件版本、组件均可查看对应扫描状态与详细信息

## 软件列表结构说明

软件管理采用三级树形结构：

- 第一级：软件
- 第二级：软件版本
- 第三级：版本组件

结构示例如下：

```text
Spring Boot
├── v3.5.12
│   ├── spring-boot-starter
│   ├── spring-core
│   └── spring-context
├── v3.4.8
│   ├── spring-boot-starter-web
│   └── spring-jdbc
└── v2.7.18
    ├── spring-beans
    └── spring-aop
```

操作路径：

```text
软件列表
  → 点击软件名称进入版本列表
    → 点击版本行「查看组件」
      → 进入组件列表
```

## 软件列表

### 1、软件列表

软件列表用于展示当前已创建的所有软件信息。点击软件名称可进入该软件的版本列表，查看该软件下的所有版本与组件信息。

点击「编辑」后，可修改软件名称、编程语言。修改后，该软件下所有版本会同步更新对应信息。

![sm-list-1](../images/sm-list-1.png)

### 2、新增软件

点击软件列表右上角「新增软件」按钮即可创建软件。新增软件时需填写软件基础信息、参考信息及软件文件，建议尽量补充完整信息，便于后续展示、检索与扫描分析。

支持上传：ZIP、JAR、WAR、TAR、工程镜像及其他容器镜像。

> 软件文件与组件文件上传后不支持修改，如需重新上传，请删除后重新创建。

### 3、版本列表

在软件列表中点击软件名称，即可进入版本列表。版本列表用于管理当前软件下的所有版本信息。

点击「新增版本」后，可继续为当前软件新增其他版本，填写信息与新增软件基本一致。

> 删除版本后，其下所有组件会同步删除，请谨慎操作。

![sm-list-2](../images/sm-list-2.png)

### 4、组件列表管理

在版本列表中点击「查看组件」，即可进入组件列表页面。

组件列表用于管理当前版本下的所有组件信息，包括组件名称、组件版本、GroupID、编程语言、开源许可证及扫描状态。组件扫描状态独立于软件扫描状态。

![sm-list-3](../images/sm-list-3.png)
![sm-list-4](../images/sm-list-4.png)

### 5、新增组件

在组件列表页面点击「新增组件」按钮即可创建组件。新增组件时需填写组件相关信息，并上传对应组件文件（支持上传：ZIP、JAR、WAR、TAR、工程镜像及其他容器镜像），填写完成后点击保存即可新增成功。

> 以下字段在组件创建成功后不支持修改，请仔细确认：组件名称、主语言、版本、GroupID、组件文件。

## 扫描状态

软件与组件均支持安全扫描。扫描内容包括：漏洞检测、License识别、风险组件分析、组件解析、来源识别。

软件列表中的扫描状态分为：

| 状态 | 说明 |
|---|---|
| 未扫描 | 当前软件暂无扫描成功版本 |
| 已扫描 | 当前软件至少存在一个扫描成功版本 |

![sm-list-5](../images/sm-list-5.png)

说明：

- 只要该软件下存在一个扫描成功的版本，软件列表即显示“已扫描”；
- 若所有版本均未扫描或扫描失败，则显示“未扫描”；
- 软件/组件扫描针对新增软件、新增版本或新增组件时上传的文件；

## 常见问题

Q：为什么软件列表显示未扫描，但版本列表显示已扫描？

A：软件列表的扫描状态汇总规则是只要有一个版本扫描成功即显示已扫描。如无扫描成功的版本，则显示未扫描。

Q：新增组件时提示某些字段不可修改怎么办？

A：组件名称、主语言、版本、GroupID和组件文件在创建后不可修改，请在保存前仔细核对。

Q：删除软件会删除其下的所有版本和组件吗？

A：会。删除软件会级联删除该软件的所有版本和组件，请谨慎操作。

Q：扫描失败如何处理？

A：请检查上传的软件文件格式是否正确（支持ZIP/JAR/WAR或工程镜像）。
