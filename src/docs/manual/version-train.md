# 版本火车

## 功能概述

版本火车用于将可信软件版本与组件打包为一个可信版本集合，按计划统一发布给组织成员使用。通过版本火车，团队可以对软件版本和组件进行集中管理、批量上架/下架，并制定发布计划，确保对外输出的开源软件版本是经过治理和评审的。

> 注：仅组织管理员可以创建、编辑、发车和管理版本火车。普通成员在管理员发车后，才能查看被授权的火车及其中的软件和组件。

### 角色权限对比

| 角色 | 权限 |
|------|------|
| 组织管理员 | 支持管理版本火车（新建、编辑、删除、发车、撤回）、查看全量的软件库和组件库 |
| 组织普通成员 | 仅支持查看已发车且有查看权限的版本火车中的软件和组件 |

## 火车列表

进入组织详情页的「版本火车」标签页，左侧展示当前组织下所有版本火车。

> 注：该页面仅限管理员查看与使用，普通成员无权限

每条火车包含以下信息：

| 字段 | 说明 |
|------|------|
| 火车名称 | 用户自定义的名称 |
| 版本号 | 该版本火车对应的版本标识 |
| 包含软件数 | 该火车下已添加的软件版本和组件总数 |
| 状态 | 当前运行状态 |

- 支持按火车编号或名称进行搜索。

![version-train-1](../images/version-train-1.png)

## 火车状态

版本火车有三种状态：

| 状态 | 说明 |
|------|------|
| 待发车 | 已创建但尚未发布，可编辑、删除、导入软件/组件 |
| 已发车 | 已发布上线，组织成员可查看和使用 |
| 已过期 | 已超过设定的结束时间，火车内软件及组件下架，组织成员无法查看 |

## 成员查看权限

版本火车支持指定可见范围，精确控制哪些普通成员可以查看该火车的内容。

![version-train-2](../images/version-train-2.png)

### 可见范围设置

在发车计划窗口中，可选择以下可见范围：

| 可见范围 | 说明 |
|---------|------|
| 全部成员 | 组织内所有普通成员均可查看该火车中的软件和组件 |
| 指定成员 | 仅被选中的特定普通成员可以查看，管理员默认始终可见 |

选择「指定成员」后，会弹出成员选择窗口，支持：

- 按成员姓名或邮箱搜索
- 批量勾选多个成员
- 查看已选成员列表

### 编辑可见范围

- 待发车和已发车的版本火车均支持编辑可见范围。已发车的火车需要先撤回发车，调整可见范围后重新发车生效。
- 已过期的火车仅支持查看和复制，不可修改可见范围。

### 权限说明

- 组织管理员始终可以查看和管理所有版本火车，不受可见范围限制
- 普通成员只能查看已发车且有查看权限的版本火车中的软件和组件
- 复制火车时会继承源火车的可见范围设置
- 如果成员退出组织，会直接从可见范围中去除

## 管理版本火车

### 新建火车

点击左侧顶部的「新增」按钮，在弹出的窗口中填写：**火车名称**（自定义名称）、**版本号**（必填，如 vt-2026-06）。

![version-train-3](../images/version-train-3.png)

### 编辑火车

仅「待发车」状态的火车支持编辑。点击火车右侧的「···」菜单，选择「编辑」即可修改火车信息。

### 删除火车

仅「待发车」状态的火车支持删除。点击火车右侧的「···」菜单，选择「删除」即可删除该火车。

## 软件管理

### 添加软件/组件到火车

在火车详情页，点击「从软件/组件库导入」按钮，弹出选择窗口。支持按「软件」和「组件」两个标签页切换，分别从软件库和组件库中选择。

- 支持按名称搜索
- 支持单条勾选和当前页全选
- 支持分页查看
- 选中后点击「确认添加」将所选条目加入火车

![version-train-4](../images/version-train-4.png)

### 软件列表

火车详情页右侧展示该火车下的软件/组件列表，包含以下字段：

| 字段 | 说明 |
|------|------|
| 软件名称 | 软件或组件的名称 |
| 版本号 | 对应的版本 |
| 语言 | 编程语言 |
| 许可证 | 开源许可证类型 |
| 状态 | 当前上架状态 |
| 操作 | 移除/上架/下架 |

列表支持按软件名称和版本号进行搜索，并支持分页。

### 软件状态

| 火车状态 | 软件状态 | 说明 |
|---------|---------|------|
| 待发车 | 待上架 | 尚未发布的软件 |
| 已发车 | 已上架 | 已对外发布，成员可查看 |
| 已发车 | 已下架 | 已从发布中移除 |

### 批量操作

选中软件后，顶部会显示批量操作栏，支持：

- **批量移除**（待发车状态）—— 从火车中移除选中的软件
- **批量上架**（已发车状态）—— 将选中的已下架软件重新上架
- **批量下架**（已发车状态）—— 将选中的已上架软件下架

![version-train-5](../images/version-train-5.png)

### 单条操作

- **移除**（待发车状态）—— 从火车中移除该软件
- **上架/下架**（已发车状态）—— 切换该软件的上架状态

## 版本火车信息树

点击「版本火车信息树」按钮，可切换展示火车中所有软件的层级结构。

信息树按以下层级组织：

```
软件名称
  └─ 版本号
       ├─ 组件 A
       ├─ 组件 B
       └─ 组件 C
```

未匹配到依赖关系的组件归类到「无依赖关系」节点下。支持「全部展开」和「全部收起」操作。

![version-train-6](../images/version-train-6.png)

## 发车

仅「待发车」状态的火车支持发车。点击「发车」按钮，设置发车计划：

| 字段 | 说明 |
|------|------|
| 开始时间 | 火车正式发布的时间 |
| 结束时间 | 火车下线的计划时间 |
| 发车备注 | 发车说明 |
| 可见范围 | 全部成员或指定成员（详见成员查看权限章节） |

发车成功后，火车状态变为「已发车」，火车中的软件默认全部为「已上架」状态，有权限的组织成员可查看并使用。

## 撤回发车

仅「已发车」状态的火车支持撤回。点击「撤回发车」按钮并确认后，火车状态恢复为「待发车」，开始时间和结束时间被清除，可重新编辑和发车。撤回后可见范围设置保留，可重新调整。

## 复制火车

已发车或已过期的火车支持复制。点击「复制火车」按钮，会基于当前火车的数据创建一个新的版本火车（状态为待发车），名称和版本号会标记为副本，可在此基础上进行编辑调整后重新发车。复制时会继承源火车的可见范围设置，可在发车前重新调整。

## 发车历史

点击左侧的「发车历史」按钮，可查看所有已发车的历史记录。

历史记录包含以下信息：

| 字段 | 说明 |
|------|------|
| 火车编号 | 系统唯一标识 |
| 名称 | 火车名称 |
| 开始时间 | 发车时间 |
| 结束时间 | 计划结束时间 |
| 状态 | 当前状态 |
| 软件数 | 包含的软件数量 |
| 组件数 | 包含的组件数量 |
| 可见范围 | 全部成员或指定成员 |

支持按火车名称搜索历史记录。

## 常见问题

**Q：发车后还能修改火车中的软件吗？**

A：已发车的火车不支持增删软件，但可以调整软件的上架/下架状态。如需修改内容，建议撤回发车或复制火车后重新编辑。

**Q：撤回发车会对已使用的成员造成影响吗？**

A：撤回发车后，火车恢复为待发车状态，成员将无法查看该火车中的软件，建议在撤回前通知相关人员。

**Q：已过期的火车还能使用吗？**

A：已过期的火车仅支持查看和复制，不支持编辑、发车或撤回操作。如需继续使用，可通过「复制火车」生成新火车后发车。

**Q：指定成员后，被指定的成员在哪里查看火车？**

A：被指定的成员登录后，在组织详情页的「版本火车」标签页中，可以看到自己被授权查看的火车，操作方式与普通火车一致。

**Q：如何修改已发车火车的可见范围？**

A：需要先撤回发车，在待发车状态下重新调整可见范围，再次发车后生效。管理员始终可以查看所有火车，不受可见范围限制。

**Q：复制火车时，新火车会继承原火车的可见范围设置吗？**

A：会继承。复制生成的新火车会保留源火车的可见范围设置，可在发车前根据需要进行调整。
