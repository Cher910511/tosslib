# 组件库

## 页面概述

在 **组件库** 中，可实现对开源软件下细分组件（Artifact/Package）的精准检索、依赖路径追踪与合规审计。

> 相较于软件库，**组件库专注于开源软件下细分交付制品的治理**，支持基于 `groupId`、`artifactId` 及 PURL 的组件检索、依赖路径分析与许可证合规审计，帮助团队在更细粒度上完成供应链资产管理与风险排查。

## 组件列表

**页面顶部**提供组件搜索与筛选入口，支持通过组件名称、组件版本及编程语言等条件进行精准检索，快速定位目标组件。

**页面中心区域**以结构化列表形式展示组件元数据，字段说明如下：

| 字段 | 说明 |
| :--- | :--- |
| **组件名称** | 组件名称（对应具体制品名称） |
| **组件版本** | 支持点击版本号进入组件详情页，查看详细信息 |
| **groupId** | 用于标识组件所属组织或发布源 |
| **编程语言** | 展示组件主要使用的开发语言 |
| **开源许可证** | 展示组件当前版本声明的开源许可证类型 |

## 组件详情

在组件列表中点击具体版本后，可进入组件详情页。

### 1. 版本信息

版本信息模块用于展示组件的基础属性、归属关系及制品信息。

* **基本信息** — 展示组件编码、`groupId`、`artifactId`、组件版本、编程语言及官方发布日期等基础数据；
* **所属软件** — 展示该组件所属的软件名称及软件版本，帮助完成从组件到软件生态的关联溯源；
* **License 信息** — 展示组件的许可证名称、许可证编码、版本及协议类型；
* **实体包信息** — 提供对应源码包或二进制包的下载地址，支持直接下载。

![component-library-2.png](../images/component-library-2.png)

### 2. 依赖路径

依赖路径模块用于分析组件在供应链中的引用关系与引入来源。

* 展示组件名称、组件版本及引入类型等信息；
* 支持识别组件是被直接引入还是通过间接依赖链传递引入，帮助研发人员排查依赖冲突与构建风险。

![component-library-3.png](../images/component-library-3.png)

### 3. SBOM

SBOM 模块用于展示组件的标准化物料清单信息，提升组件资产透明度。

* 规范展示 `groupId`、`packageName`、`version`、编程语言等字段；
* 提供标准化的 **PURL** 唯一标识，为企业自动化资产盘点、依赖治理与供应链审计提供底层数据支撑。

![component-library-4.png](../images/component-library-4.png)

> * `groupId`：组件所属组织或发布方标识，通常用于区分组件来源。

> * `artifactId`：组件制品名称，用于标识具体模块或功能包。

> * `PURL（Package URL）`：软件包标准化唯一标识，用于统一描述不同生态中的组件信息，便于供应链追踪与资产管理。
